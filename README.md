## 🎯 الفكرة

عند أي صفقة (عقار، مركبة، نشاط تجاري…) يُدفع **العربون** لإثبات الجدية — لكن يبقى السؤال:
**من يضمن هذا المبلغ؟** اليوم يُسلَّم العربون يداً بيد بلا طرف محايد، فتكثر النزاعات:
البائع يماطل، أو المشتري ينسحب، أو يختلفان على الاسترداد.

**عربون** يحل هذا بأن يكون **الطرف المحايد الموثوق**:

> يُودَع مبلغ العربون في **حساب ضمان مستقل (Escrow)** ولا يُفرَج عنه إلا وفق شروط
> العقد الموقّع رقمياً بين الطرفين — فلا أحد يستطيع الاستيلاء على المال أو إنكاره.

المبدأ الأساسي في المنصّة: **حقّ المشتري محفوظ** — يستطيع الانسحاب قبل إتمام الصفقة
واسترداد كامل عربونه، دون أي مصادرة.

## ✨ أبرز الميزات

- 🏦 **حساب ضمان لكل صفقة** — يُحتجز العربون بأمان حتى الإتمام.
- 📝 **عقود رقمية موثّقة** — قوالب جاهزة (عقار/مركبة/نشاط تجاري/عام) بتوقيع الطرفين وطابع زمني.
- 🔄 **سوق التنازلات** — إمكانية التنازل عن صفقة قائمة لمشترٍ آخر عبر الضمان.
- ⚖️ **إدارة النزاعات** — فتح نزاع يُجمّد الأموال فوراً حتى صدور القرار.
- 📊 **لوحة متابعة** — كل الصفقات وحالاتها والمبالغ المحتجزة في مكان واحد.
- 🔔 **إشعارات وتتبّع زمني** — سجل كامل لكل حدث في الصفقة.
- 🆔 **توثيق الهوية (نفاذ)** — لا عملية مالية قبل التحقق من الهوية.

## 🔐 النواة المصرفية (Banking Core)

بُنيت المنصّة على ضوابط بمستوى الأنظمة البنكية:

| الضابط | الوصف |
|---|---|
| **دفتر قيد مزدوج** | كل حركة مال تُسجَّل كقيود مدين/دائن متوازنة، غير قابلة للتعديل، مع مطابقة دورية (مجموع المدين = مجموع الدائن). |
| **حالة أموال مستقلة** | لكل صفقة حالة أموال منفصلة: `بانتظار الإيداع → محتجز → مُحرَّر / مُسترَد`. |
| **رقابة مزدوجة (Maker-Checker)** | أي تحرير/استرداد للأموال يتطلّب طلباً من موظّف واعتماداً من مدقّق **مختلف** — لا أحد يحرّك المال وحده. |
| **آلة حالات صارمة** | لا تنتقل الصفقة إلا بين حالات مسموح بها؛ يُرفض أي قفز غير مشروع. |
| **تجميد عند النزاع** | فتح نزاع يُجمّد الأموال ويمنع أي تحريك حتى الحسم. |
| **بوّابة KYC** | لا تُنفَّذ أي عملية مالية قبل توثيق الهوية. |

## 🔄 رحلة الصفقة

```
تسجيل + توثيق الهوية
        ↓
إنشاء صفقة واختيار قالب العقد
        ↓
توقيع الطرفين رقمياً
        ↓
إيداع العربون في حساب الضمان  →  [ الأموال محتجزة ]
        ↓
   ┌───────────────┬──────────────────┐
   ↓               ↓                  ↓
إتمام الصفقة    انسحاب المشتري       نزاع
تحرير للبائع    استرداد كامل        تجميد ثم قرار
+ رسوم المنصة   للمشتري            (استرداد / تسوية)
```

## 🧩 البنية التقنية

منصّة أحادية المستودع (monorepo) على pnpm workspaces:

| الحزمة | الدور |
|---|---|
| `artifacts/arboun` | الواجهة الأمامية — React 19 + Vite + Tailwind + wouter |
| `artifacts/api-server` | خادم الـ API — Express 5 (يقدّم الـ API + الواجهة المبنية) |
| `lib/db` | مخطّط قاعدة البيانات — Drizzle ORM + PostgreSQL |
| `lib/api-spec` · `lib/api-zod` · `lib/api-client-react` | عقد الـ API المولّد (OpenAPI → Zod + React Query) |
| `scripts` | سكربتات التشغيل والبيانات التجريبية |

**التقنيات:** TypeScript · React 19 · Express 5 · PostgreSQL + Drizzle · Zod · esbuild · Docker · DigitalOcean.

### التشغيل محلياً

```bash
pnpm install
export DATABASE_URL="postgres://…"        # قاعدة PostgreSQL
export PORT=8080 BASE_PATH=/ SESSION_SECRET=dev
pnpm --filter @workspace/arboun run build # بناء الواجهة
pnpm --filter @workspace/api-server run dev
```

> يُنشئ الخادم الجداول تلقائياً عند الإقلاع، ويحمّل بيانات تجريبية إذا كانت القاعدة فارغة
> (يمكن تعطيلها بـ `DEMO_SEED=false`).

### النشر

يُبنى المشروع عبر `Dockerfile` ويُنشر على **DigitalOcean App Platform** (نشر تلقائي مع كل دفع على `main`).
المتغيّرات المطلوبة: `DATABASE_URL` · `SESSION_SECRET`.
